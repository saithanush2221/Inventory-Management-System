# Inventory Management System - Modern Inventory Management System

![InvenTrack Banner](https://via.placeholder.com/1200x400/0f172a/ffffff?text=InvenTrack+Analytics+%26+Inventory)

InvenTrack is a comprehensive, full-stack, enterprise-grade inventory management system designed to streamline supply chain operations, track sales, manage suppliers, and provide real-time analytical dashboards.

## 🌟 Key Features

* **Real-time Analytics Dashboard:** Beautiful charts (Recharts) for sales trends, inventory valuation, and low stock alerts.
* **Product & Inventory Management:** Track stock levels across multiple warehouses, manage SKUs, and view live inventory logs.
* **Order & Invoice Processing:** Create sales orders, generate invoices, and manage statuses (Pending, Shipped, Delivered).
* **Supply Chain:** Manage suppliers and generate Purchase Orders for seamless stock replenishment.
* **Customer Management:** Maintain customer profiles and track their order histories.
* **Secure Authentication:** Role-based access control with robust JWT (JSON Web Token) authentication.
* **Modern UI/UX:** Responsive, dynamic interface with glassmorphism aesthetics, dark mode, and micro-animations.

## 🛠️ Technology Stack

**Frontend (Client)**
* [Next.js](https://nextjs.org/) (React Framework)
* [Tailwind CSS](https://tailwindcss.com/) (Styling)
* [Recharts](https://recharts.org/) (Data Visualization)
* [Lucide React](https://lucide.dev/) (Icons)

**Backend (Server)**
* [Node.js](https://nodejs.org/) & [Express.js](https://expressjs.com/) (API)
* [Prisma](https://www.prisma.io/) (Next-generation ORM)
* [SQLite](https://www.sqlite.org/) (Database - Easily swappable to PostgreSQL)
* [JSON Web Tokens](https://jwt.io/) (Authentication)

## 🚀 Getting Started Locally

Follow these instructions to get a copy of the project up and running on your local machine for development and testing.

### Prerequisites

Ensure you have the following installed:
* **Node.js** (v18 or higher)
* **npm** or **yarn**
* **Git**

### 1. Clone the repository
```bash
git clone https://github.com/saithanush2221/Inventory-Management-System.git
cd Inventory-Management-System
```

### 2. Setup the Backend
Open a terminal and navigate to the backend directory:
```bash
cd backend
npm install
```

Configure your environment variables by creating a `.env` file in the `backend` folder:
```env
DATABASE_URL="file:./prisma/dev.db"
JWT_ACCESS_SECRET="supersecretaccess_key_32chars_min"
JWT_REFRESH_SECRET="supersecretrefresh_key_32chars_min"
PORT=4000
```

Initialize the database and seed it with mock data:
```bash
npx prisma db push
npx prisma generate
npx ts-node prisma/seed.ts
```

Start the backend server:
```bash
npm run dev
```
*The backend API will run at `http://localhost:4000/api`*

### 3. Setup the Frontend
Open a new terminal and navigate to the frontend directory:
```bash
cd frontend
npm install
```

Start the frontend development server:
```bash
npm run dev
```
*The web interface will run at `http://localhost:3000`*

### 4. Login Credentials
Once the frontend is running, you can log in using the mock data generated by the seed script:
* **Email:** `admin@inventory.com`

## ☁️ Deployment

This project is built to be easily deployed to modern cloud providers:

* **Frontend:** Deployed seamlessly on [Vercel](https://vercel.com). Just connect your GitHub repo, set the Root Directory to `frontend`, and configure the `NEXT_PUBLIC_API_URL` environment variable.
* **Backend:** Deployed on [Render](https://render.com). Set the Root Directory to `backend`, set the build command to `npm install && npx prisma generate && npm run build`, and configure your JWT environment variables.
